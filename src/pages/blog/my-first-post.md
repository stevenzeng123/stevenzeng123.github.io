# Building My Portfolio Site with AI

When I started building this site, I did not approach it like a traditional web project. Instead of planning every component and writing everything manually, I used what people often call **“vibe coding”**: iterating quickly with the help of an AI assistant while focusing on design direction and technical decisions.

The goal was not just to build a portfolio, but to experiment with a workflow where AI helps accelerate development while I still guide the architecture.

## Choosing the Stack

After comparing several tools, I chose **Astro** as the static site framework. Astro provides a clean structure for static websites while still supporting modern development practices.

The site uses:

- **Astro** for static site generation
- **Tailwind CSS** for styling
- **Markdown** for blog content
- **GitHub Pages + GitHub Actions** for deployment

This stack keeps the site lightweight while still allowing flexible layout and styling.

## Building with AI Assistance

Throughout development, I used an AI assistant to speed up several tasks:

- generating layout templates
- refining the homepage structure
- improving resume bullet points
- configuring GitHub Pages deployment
- debugging configuration issues

Rather than copying code blindly, I treated the AI as a **development partner**. I would describe the goal (for example, designing a minimal homepage layout), review the suggested code, and then modify it to fit the design of the site.

This allowed me to iterate quickly without getting stuck on small implementation details.

## Iterating on Design

The biggest challenge was maintaining a **consistent visual style** across the site.

Early versions of the blog page looked disconnected from the rest of the layout. After several iterations, I introduced a shared layout system so that pages inherit the same navigation, spacing, and typography.

Small changes like consistent containers, subtle borders, and a blue accent color helped the site feel more polished.

## What I Learned

This project showed that AI tools can dramatically speed up development, especially for repetitive tasks and boilerplate code.

However, they still require careful guidance. The most effective workflow was:

1. Define the design or technical goal.
2. Generate a starting point with AI.
3. Refine and adapt the code manually.

In other words, **AI works best when used as a collaborator, not a replacement for understanding the system.**

## Final Thoughts

Vibe coding this site was an interesting experiment in AI-assisted development. It allowed me to focus more on architecture and design decisions while still moving quickly.

As AI tools improve, workflows like this will likely become a normal part of building small web projects and prototypes.
